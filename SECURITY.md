# Security Policy

## Supported version

Security fixes are applied to the latest commit on `main`.

## Report a vulnerability

Do not open a public issue with vulnerability details, credentials, browser
state, screenshots, or proof-of-concept data.

Use GitHub's private vulnerability reporting for this repository. Include the
affected commit, impact, reproduction steps, and any suggested mitigation.

Before making the repository public, the maintainer must enable **Private
vulnerability reporting** under **Settings → Security → Code security**.

If private reporting is temporarily unavailable, open a minimal issue asking
the maintainer to enable a private reporting channel without including security
details.
